# Security Policy

Mortar is early in development and not yet production-ready.

## Reporting a vulnerability

Please do not open a public issue for suspected security vulnerabilities.

For now, report security concerns privately to the project maintainer through a non-public channel before any public disclosure. If a dedicated reporting address or process is added later, this file should be updated.

## Scope

Security reports are especially relevant for:

- authentication and session handling
- secret management and config interpolation
- plugin proxying behavior
- exposure of upstream service credentials or user data

## Expectations

- Include reproduction steps when possible
- Give the maintainer time to investigate before public discussion
- Avoid including real secrets or personal data in the report
