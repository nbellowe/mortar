# Documentation Site — 2026-05-08

## What was built

A VitePress documentation site in `docs/site/`, deployed to GitHub Pages via a GitHub Actions workflow (`.github/workflows/docs.yml`). Published at `https://nbellowe.github.io/mortar/`.

## Audience structure

Three distinct sections, each with its own sidebar:

- **Users** — plain language, task-oriented (request media, browse library). No homelab jargon; readers are not expected to know what Sonarr or SABnzbd is.
- **Operators** — technical readers deploying and configuring Mortar. Covers installation, the config YAML format, and per-plugin setup.
- **Contributors** — dev setup, architecture overview, and a guide to writing new plugins.

## Key decisions

**VitePress over Jekyll/MkDocs/Docusaurus.** The project already uses Node/npm (Expo). VitePress is minimal, TypeScript-native, and produces a fast static site without a heavy framework. Jekyll would have worked without a build step but brings a Ruby dependency; Docusaurus would have been heavier than needed for the current content volume.

**Separate `package.json` in `docs/site/`.** Keeps docs tooling isolated from the Expo app's dependency tree. The Expo app and the docs site are built and deployed independently.

**GitHub Actions native deployment.** The workflow uses `actions/deploy-pages` rather than pushing to a `gh-pages` branch. This requires setting Pages source to "GitHub Actions" once in repo settings — no further manual steps after that.

**Workflow triggers on `docs/site/**` path filter.** A push to `main` that touches only backend or frontend code does not trigger a docs rebuild. This keeps CI fast.

**`v0.x` framing in operator docs.** Installation docs note that pre-built binaries and Docker images aren't published yet. This is accurate and avoids misleading operators who find the site before those artifacts exist.

## What was deferred

- A logo (`/logo.svg` is referenced in the config but not yet created)
- Notification docs (notifications are post-v1 per the vision spec)
- Native client docs (experimental during v0.x, not release-blocking)
- A search index beyond VitePress's built-in local search
